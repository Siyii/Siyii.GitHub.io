成为一名优秀前端开发工程师的 8 个技巧建议 （上）

我应该用这个插件还是那个库来实现呢？

我应该先规划好整个项目还是直接撸起袖子开始码代码呢？

我应该向别人寻求帮助还是自己冥思苦想呢？

这些问题是不是似曾相识？

我和你感同身受。不过当我将下文中提到的技巧应用到我的日常工作中，一切问题都迎刃而解。

不管你是刚入行的菜鸟还是行家，这些技巧都将会帮你跃上职业生涯约的新台阶。

1. 做好项目规划

请注意，译者这里将 plan 译为规划，而不是计划。这两者有明显的区别：规划是宏观的，需要考虑到事物的全局性，会包含多个可执行的计划；而计划是微观的，灵活且具有操作性的，是拆分到颗粒度最小的待办事项。

即使对于大多数资深工程师而言，事先不做好规划仍然是一个非常常见的问题。

规划是无聊的，我们并不知道如何去做规划，并且我们会觉得这不是我们的工作。

这也可能是我们经常性忽略这一步的主要原因。

好的规划能帮助你：

  • 节省时间
  • 当 deadline 逼近时缓解你的压力
  • 每天能轻松地准时完成工作

做规划本不是一件复杂的事情，制作一个简单的项目 to-do-list 可以让你

  • 保持工作重心
  • 保持头脑清醒，专注于手头上的事情
  • 在项目早起阶段就识别出阻力
  • 让你与你的客户、上司或者项目经理更简单地保持信息对齐

2. 积极主动

没有什么事情会比告诉你的客户或老板，项目要比预期花费更长时间，项目不能如期交付，更糟糕的事情了。

有一个用有的办法是，积极主动地同步项目中遇到的阻力，可能的延期等等项目风险。

项目花费的事情比预期计划的时间要长是经常会发生的，但是你不能做到只有自己知道风险，而你的上司却不知道。

你碰上了一个 bug 一点头绪都没有？你并不孤单，每个人甚至是最资深的工程师几乎在每个项目中都会碰到同样的问题。

如果碰上了这样的问题，记得及时寻求帮助！

每个人都会理解你尽了自己最大的努力，并且让他们知道了你目前的进度。

设置一个每日或者每周的提醒事项，来帮助及时提醒自己做好项目进度的更新。

3. 保持好奇

我认为保持好奇在这几个技巧中是最为重要的一个。

几乎每天都有新的工具，插件，技术和编程语言在发布。

对新的事物保持开发的心态是尤为重要的，能让你有动力去学习和理解、应用他们。

前端技术发展之美，或许是最糟糕的，就是发展得非常快，有很多新的事物涌进来。

这意味着你可能会很快成长为技术专家，但也可能会快速被淘汰。

建议：你并不需要知道一切，对于一个比较新的领域或者一门技术你可以浅尝遏止。当这个技术发展得比较成熟的事情，就可以去深入学习。

一些有用的资源：（译者：下面的资源都是英文的，建议多多食用~）

  • CSS Weekly
  • HTML5 Weekly
  • JavaScript Weekly
  • Responsive Design Weekly

4. 技能投资

任何和前端领域有关的都可以被视为一项技能，并且是可以习得的。

为了在前端领域有持续的成长，你将会需求投资自己。我并不是让你把钱花在昂贵的课程，讲座和书上，我更赞同的是投资你的时间。

通往成功的路上没有捷径，调查显示，要彻底掌握一门技能要花费 10000 个小时，这意味着要一周练习 40 小时，持续 4 年。

做好准备，在无数个深夜中，查阅学习他人的网站，使用新发布的插件和框架，阅读 web 发展相关的博客，收听博客，咀嚼成千上万条信息。

建议：每周安排一段时间进行自我提升。我知道这可能很难做到，如果你的老板不支持你的话。不过一定要为自己的职业生涯投资一些时间。

总结（译者的小摘 O(∩_∩)O~）：

  • 项目开始时，做好全局的规划，及时识别风险；每天的工作内容要列好 to-do-list。
  • 及时和你的上司、项目组成员同步进度和风险，不要拖到结束时才去暴露，否则会被视为工作能力不行 😫
  • 了解前端最新的资讯，不要求精通，但是也要丰富自己的技术视野、不被老旧的观念束缚。
  • 空闲时间投资自己！任何和工作有关的都可以去投资自己，比如项目管理、风险管理、设计、编程能力等等，包含了硬实力和软实力。
